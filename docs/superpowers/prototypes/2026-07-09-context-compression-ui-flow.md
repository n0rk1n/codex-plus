# Context Compression UI Flow Prototype

日期：2026-07-09

## 状态

本文件是上下文压缩功能的 UI/交互基准。它不新增需求，专门把已批准的设计和实现计划转译成界面状态、点击路径、视觉标记和人工验收脚本。

关联文档：

- `docs/superpowers/specs/2026-07-08-context-compression-design.md`
- `docs/superpowers/plans/2026-07-09-context-compression.md`

## 设计目标

用户在主时间线中始终优先看到原文，同时能一眼识别哪些轮次影响了模型输入。压缩、修改、排除、失败、回滚和拼接关系不应把界面变成技术调试器；它们应该以轻量标记、清晰边界、右侧历史面板和模型输入预览来解释。

核心体验判断：

- 用户能知道“我看到的原文”和“发给模型的内容”哪里不同。
- 用户能恢复查看任意原始轮次。
- 用户能追溯每个压缩版本从哪些轮次或版本生成。
- 用户能明确看到当前最终发给模型的是哪条活跃链路。
- 上下文超限时，用户知道为什么不能发送，以及如何交给系统压缩后继续。

## 第一屏布局

主 Workbench 仍保持当前 V1 悬浮工作台结构：

- 顶部：项目/对话选择和全局操作。
- 中部：原文时间线。
- 底部：输入框、预算状态、发送/停止按钮。
- 右侧：默认隐藏的压缩历史 inspector，仅在用户查看历史、编辑、压缩、预览时展开。

压缩功能不是一个独立页面。它是对当前对话时间线的增强。

## 时间线状态

### 原文发送

含义：这一轮没有活跃压缩、修改或排除。模型输入使用原始用户段和 AI 段。

视觉：

- 原文正常显示。
- 轮次边缘没有压缩边界。
- 鼠标悬停时可显示极轻量状态 `原文发送`。
- 不主动占用视觉注意力。

操作：

- 用户可以选择该轮次。
- 选择后出现轻量操作条：`编辑`、`压缩`、`排除`、`预览模型输入`。

### 已修订

含义：用户手动修改了一个用户段或一个 AI 段，模型输入使用修订版本。

视觉：

- 原文仍显示在主时间线。
- 被修订段旁出现细边界标记。
- 状态标记显示 `已修订`。
- 悬停摘要展示：修订对象、修订时间、当前活跃版本摘要。

操作：

- 点击标记打开 popover。
- Popover 中有 `查看完整历史`。
- Inspector 中展示原文、修订版本、回滚、继续压缩、预览模型输入。

### 已压缩

含义：一个轮次或连续多轮已经被压缩成活跃版本。模型输入使用压缩文本。

视觉：

- 原文保持可读。
- 压缩覆盖范围有开始/结束边界。
- 边界之间不折叠原文，避免用户误以为原文被替换。
- 状态标记显示 `已压缩`。

操作：

- 点击标记查看压缩摘要。
- Inspector 展示压缩来源、输入快照、模板、用户意图、provider、模型、输出版本。
- 可继续压缩最后活跃版本。

### 拼接压缩

含义：一个活跃版本由多个相邻轮次或多个已有版本合并压缩而来。

视觉：

- 覆盖范围显示一组连续边界。
- 被拼接的相邻块之间出现轻量连接提示。
- 当前滑到某一块时，显示它与上方或下方块存在关联。
- 主时间线只展示关系提示，不展示完整图谱。

操作：

- Popover 摘要列出“来源：第 N-M 轮”。
- Inspector 中用线性历史加辅助连接展示关系。
- 模型输入预览只显示拼好的最终文本，不显示关系图。

### 已排除模型上下文

含义：这一轮原文仍可见，但不参与模型输入。

视觉：

- 原文变淡或半透明。
- 状态标记显示 `已排除模型上下文`。
- 不使用红色警告风格，因为这是用户主动控制，不是错误。

操作：

- Inspector 中可恢复原文参与模型输入。
- 模型输入预览中该轮不出现。

### 压缩失败

含义：一次压缩尝试失败。失败记录保留在版本历史中，但不能成为活跃版本。

视觉：

- 原文不变暗、不折叠。
- 状态标记显示 `压缩失败`。
- 标记比成功压缩更克制，避免误导用户以为当前模型输入已经改变。

操作：

- Popover 展示错误摘要。
- Inspector 展示失败输入来源、provider、错误信息。
- 用户可以重新压缩或改用手动编辑。

## 状态标记层级

主时间线只承担快速识别，不承担完整解释。

层级顺序：

1. 边界线：说明哪里开始、哪里结束。
2. 状态短标签：说明当前参与模型输入的状态。
3. Popover：说明当前块的摘要和入口。
4. Inspector：说明完整历史、来源、版本、回滚和后续操作。
5. 模型输入预览：说明最终发给模型的文本。

## 右侧历史 Inspector

Inspector 是压缩功能的主要解释面板。

默认结构：

- 顶部：当前选择范围和活跃状态。
- 活跃版本：展示当前发给模型的版本摘要。
- 历史列表：按时间线性展示 original、manual edit、compression、system compression、failed、rollback、tombstone。
- 来源映射：展示该版本来自哪些轮次或版本。
- 拼接关系：当版本来自多个相邻块时展示轻量连接。
- Provider 信息：展示压缩 provider、模型、模板、用户补充意图、创建时间。
- 操作区：编辑、继续压缩、回滚、排除、恢复原文、预览模型输入。

Inspector 不做成图数据库式界面。默认线性历史优先；只有拼接关系需要辅助图形提示。

## 点击路径

### 查看某一轮历史

```text
时间线状态标记
  -> Popover
  -> 查看完整历史
  -> 右侧 Inspector 展开
```

结果：

- 主时间线保留原位置。
- Inspector 选中对应轮次或范围。
- 对应原文范围轻微高亮。

### 手动编辑 AI 段

```text
选择一轮
  -> 操作条：编辑
  -> 选择 AI 段
  -> 编辑弹窗
  -> 保存
  -> 创建 manual_edit 版本
  -> 刷新标记为 已修订
```

规则：

- 只能编辑一个用户段或一个 AI 段。
- 允许保存为空字符串。
- 原文仍在主时间线显示。
- 模型输入使用最新活跃修订版本。

### 默认压缩连续多轮

```text
拖选或 Shift 选择连续轮次
  -> 操作条：默认压缩
  -> 后台 Codex CLI 压缩
  -> 成功：创建 default_compression 版本并激活
  -> 失败：创建 failed_compression 历史记录，不改变活跃链路
```

规则：

- 选择必须是连续轮次。
- 每一轮都按“用户段 + AI 段”参与压缩。
- 压缩运行不写入普通对话历史。

### 自定义压缩

```text
选择连续轮次
  -> 操作条：自定义压缩
  -> 选择提示词模板
  -> 补充本次压缩意图
  -> 开始压缩
```

结果：

- 模板 ID 和用户补充意图写入 `compression_inputs`。
- 用户补充意图不出现在普通聊天记录。
- 成功后标记为 `已压缩` 或 `拼接压缩`。

### 继续压缩最后版本

```text
打开 Inspector
  -> 选择当前活跃版本
  -> 继续压缩
  -> 选择默认或自定义压缩
  -> 新版本成为活跃版本
```

结果：

- 旧活跃版本变成历史版本。
- 新版本继承来源链路。
- 模型输入只使用最后活跃版本。

### 回滚版本

```text
打开 Inspector
  -> 选择历史版本
  -> 回滚到此版本
  -> 创建新的活跃分支
  -> 被替换的下游分支进入 tombstone
```

结果：

- 原始事件不删除。
- 被替换分支不再进入正常 UI 的活跃链路。
- Inspector 仍可显示 tombstone 摘要用于追溯。

### 排除一轮

```text
选择一轮
  -> 操作条：排除
  -> 状态变为 已排除模型上下文
```

结果：

- 原文变淡。
- 模型输入预览中不出现该轮。
- Inspector 可恢复。

### 查看模型输入

```text
Inspector 或操作条
  -> 预览模型输入
  -> Sheet 展示最终拼好的文本
```

规则：

- 只展示发给模型的最终文本。
- 不展示来源图、版本图、解释文本。
- 预览内容必须和发送时使用的 assembler 输出一致。

### 硬限制系统压缩

```text
预算状态达到 hardLimit
  -> 发送按钮禁用
  -> 显示 需要压缩后继续
  -> 显示 交付系统完成压缩
  -> 点击按钮
  -> 系统使用当前活跃模型输入进行压缩
  -> 结果加入版本控制
```

结果：

- 系统压缩不是普通聊天消息。
- 系统压缩必须记录拿了哪些轮次或版本。
- 成功后刷新预算、标记和模型输入预览。
- 失败后发送继续禁用，并在历史中显示失败记录。

## 模型输入预览

模型输入预览是一个结果视图，不是调试视图。

内容：

- 最终拼好的文本。
- Copy 按钮。
- Close 按钮。

不展示：

- 版本树。
- 来源轮次。
- Provider 元数据。
- 压缩提示词。
- 用户补充意图。

这些信息全部放在 Inspector。

## 视觉规则

- 使用 macOS 风格的轻量材质、细边界、低饱和状态色。
- 不使用大面积警告色。
- 不用卡片嵌套卡片。
- 主时间线保持阅读优先。
- 状态标签不抢正文注意力。
- Inspector 可以信息密度更高，但保持分组清楚。
- 连接关系只在需要解释拼接压缩时出现。
- 文案短、明确、面向用户行为。

## 状态文案

固定文案：

- `原文发送`
- `已修订`
- `已压缩`
- `拼接压缩`
- `已排除模型上下文`
- `压缩失败`
- `查看完整历史`
- `预览模型输入`
- `继续压缩`
- `恢复原文`
- `需要压缩后继续`
- `交付系统完成压缩`

## 人工验收脚本

使用一条包含至少 7 轮的对话，记为 `ABCDEFG`。

### 1. 原文默认可见

- 打开对话。
- 验证 A-G 原文全部可见。
- 验证没有压缩时状态不喧宾夺主。

### 2. 手动修改 AI 段

- 选择 B 轮的 AI 段。
- 只保留 AI 回复中的一句话。
- 保存。
- 验证主时间线仍显示 B 轮原文。
- 验证 B 轮状态为 `已修订`。
- 验证模型输入预览使用修改后的 B。

### 3. 默认压缩连续轮次

- 选择 C-D 连续两轮。
- 执行默认压缩。
- 验证 C-D 有开始和结束边界。
- 验证状态为 `已压缩`。
- 验证 Inspector 记录压缩输入来源为 C-D。

### 4. 拼接压缩

- 继续将已压缩 C-D 与 E 轮拼接压缩。
- 验证滑到 C-D 或 E 时能看到相邻关联提示。
- 验证状态为 `拼接压缩`。
- 验证模型输入中 C-D-E 只出现最终拼接版本一次。

### 5. 排除轮次

- 排除 F 轮。
- 验证 F 原文变淡并标记 `已排除模型上下文`。
- 验证模型输入预览不包含 F。
- 恢复 F，验证状态回到可参与模型输入。

### 6. 回滚

- 在 Inspector 回滚到 C-D 的早期压缩版本。
- 验证创建新的活跃分支。
- 验证被替换的后续分支不再进入模型输入。
- 验证原文仍可查看。

### 7. 压缩失败

- 使用一个会失败的 fake provider 或测试开关触发压缩失败。
- 验证状态显示 `压缩失败`。
- 验证失败版本不成为活跃版本。
- 验证原模型输入不被破坏。

### 8. 硬限制系统压缩

- 模拟 hardLimit。
- 验证发送按钮禁用。
- 验证显示 `需要压缩后继续`。
- 点击 `交付系统完成压缩`。
- 验证系统压缩结果进入版本历史。
- 验证系统压缩记录使用了哪些轮次或版本。
- 验证成功后发送状态重新按预算结果刷新。

### 9. 归档

- 归档对话。
- 验证归档包含原文。
- 验证归档包含压缩版本历史、失败、tombstone、来源映射、最终活跃模型输入。
- 验证归档搜索以原文为准。

## 通过标准

该 UI flow 通过的标准：

- 用户无需打开技术日志，就能知道当前模型输入和原文哪里不同。
- 用户能从任意状态进入完整历史。
- 用户能确认系统压缩拿了哪些内容。
- 用户能看到拼接压缩与相邻块的关系。
- 模型输入预览与发送内容一致。
- 超限时用户有明确下一步。
- 所有操作都能回到原文可恢复的事实基础。
